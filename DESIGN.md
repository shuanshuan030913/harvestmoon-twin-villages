# DESIGN.md — 雙子村攻略網站視覺設計語言

> 2026-07-15 定案（使用者自「牧場手帳 vs 雙村對照」兩方向 mockup 中選定方向 A）。
> 本檔是視覺工作的憲章：任何 UI 樣式的新增或修改，先讀本檔、以本檔為準；
> 本檔沒定的事，先補進本檔再實作（同 `.spec/` 規格缺口原則）。

## 視覺主題與氛圍

**牧場手帳**——整個網站是「玩家自己寫的攻略手帳」：羊皮紙底、貼紙卡片、
紙膠帶、印章圖示、拍立得頭像、楷體手寫感標題。溫暖、手作、低密度。

- 識別度押在「質感」（紙、印章、手寫），不押結構性色彩系統。
- 村色（bluebell／konohana）保留為**內容語意色**（`data-village` 機制不變），
  不承擔頁面級識別。
- 遊戲素材（`content/images/` 的頭像、截圖）是一等設計素材，能用圖就不畫圖。

## 色盤與角色定義

| Token | Hex | 角色 |
|---|---|---|
| `--color-parchment` | `#f5ead1` | 頁面底色（配 `.bg-dots` 點點紋理） |
| `--color-cream` | `#fbf4e4` | 卡片／貼紙面 |
| `--color-ink` | `#4a3728` | 文字、邊框（手繪筆觸感） |
| `--color-soil` | `#8b7355` | 次要面、紙膠帶 |
| `--color-seal` | `#b3502e` | 印泥紅：印章、「最愛」強調（省著用） |
| `--color-bluebell` | `#4a7fb5` | 藍鈴村語意色（僅標記內容歸屬） |
| `--color-konohana` | `#5a9e4b` | 此花村語意色（僅標記內容歸屬） |
| `--color-plum` | `#8d6b7a` | 「無特定村莊」卡片專用底色（U36，2026-07-22；雙村共通／無 village 欄位的 collection） |

- 低對比文字下限：`text-ink/60`；再淡只能用於裝飾性文字，不得承載資訊。
- 不引入純白、純黑、冷灰。
- `--color-plum` 是全站唯一的第 3 種「中性」色（在 ink／soil 之外），**只用在列表卡的
  村色底色**、不作強調色使用——原本雙村共通落在 soil，跟篩選 chip 的 ink 底色同色系
  深淺兩階，怎麼調透明度都拉不開差別；plum 色相跟 ink/soil（棕）、bluebell/konohana
  （藍/綠）、seal（紅）都不同，才真正解決撞色問題。

## 排版規則

| 角色 | 字體 | 用法 |
|---|---|---|
| 手寫（標題／貼紙標籤／註記） | `font-hand`（楷體 stack） | 站名、分類名、手帳註記；一律短字串 |
| 內文 | 系統黑體（預設） | 段落、表格、chips——**內文永遠不用楷體**（可讀性優先） |

- 楷體 stack：`"Kaiti TC", "BiauKai", "Kaiti SC", "KaiTi", "DFKai-SB", serif`。
  已知風險：Android 無楷體會落到 serif，視覺弱化但可讀；接受此退化，
  若日後要根治再評估中文子集 webfont（站名等短字串子集化可行）。
- 字級照 Tailwind 預設 scale，不自訂中間值。
- 長文內文（條目頁／攻略頁 `entry.html` 渲染區）用 Typography 外掛的 `prose`（非 `prose-sm`）：
  這是全站唯一連續閱讀（非掃視）的內容，手機上 14px 偏小（2026-07-18 使用者裁決）。
  **例外：手帳行（見下方三語言分工）字級固定 `text-sm`（14px）**（2026-07-19 使用者裁決）：
  手帳行是 `ul`/`li` 條列式格線（家庭關係、每日路線時間表等），視覺上與資訊列同語言，
  字級應對齊資訊列 14px；此例外只套用在 `li`，段落級的連續閱讀文字不受影響、維持較大字級。
- **`text-xs`（12px）只留給「最弱的連結」**（2026-07-18 使用者裁決追加）：原始出處外部連結、
  清除篩選這類次要動作連結。其餘會被使用者當作「內容」讀的文字——物品 chip 標籤
  （最愛／喜歡／討厭／最討厭）、篩選群組標籤與選項、存讀檔按鈕與狀態訊息、
  行事曆格內的生日/節慶連結、角色卡村莊/生日說明——一律至少 `text-sm`（14px），
  不得停在 12px。純粹次要 meta（如「N 筆」計數、首頁貼紙牆下方的條目數）仍可維持 `text-xs`。

## 元件樣式

### 貼紙卡（sticker）
分類入口、可點的卡片單位。`bg-cream`、`1.5px` 實線 ink 邊框、`rounded-[9px]`、
硬陰影 `2px 2px 0`（ink 16%）、頂端紙膠帶（`::before`）、微旋轉
（grid 內依 nth-child 輪替 -1.4° / 0.9° / -0.5°）。

### 印章（stamp）
圖示的標準容器：圓形 ink 細環＋外圈 offset 細環、內放線條圖示
（`icons.jsx` sprite，stroke 1.8、圓端點）、微旋轉。
**禁止 emoji 當圖示**——這是本次改版要消除的頭號「AI 味」。

### 紙膠帶（tape）
半透明 soil 漸層小矩形、微旋轉。一個元件至多兩條，多了像垃圾。

### 拍立得（polaroid）
角色頭像容器：白框、下方留白寫說明（楷體小字）、兩角紙膠帶、微旋轉、柔陰影
（全站唯一允許柔陰影的元件，模擬照片浮起）。
說明列＝日文名＋生日手寫日期（遊戲曆法「春の月N日」，ink 55%）——像在拍立得上手寫拍攝日期；
生日入拍立得後不再重複列於資訊列（2026-07-17 使用者裁決，與村莊印章同一原則）。
尺寸以圖片**原生像素 1:1** 為準（2026-07-19 使用者回饋：頭像被縮小反而比內文圖模糊）：
頭像皆為 DS 截圖約 254px 寬，容器 272px（含相紙邊 8px×2）＋`max-w-full`；
內文圖同理不放大超過原生（條目頁 prose img 上限 255px）。

### 印章章（seal）
村莊歸屬章：圓環 `--color-seal` 邊框、楷體直排短字（「此花」「藍鈴」）、微旋轉。

### 資訊列（journal row）
條目欄位呈現：label＋值、底部 `1.5px` 點狀虛線（`border-dotted` ink 40%），像手帳格線。
**全站統一「標題在上、值在下」單一版型**（2026-07-21 使用者裁決，廢除舊「單值左右並排」變體）：
不論欄位來自 `columns`（跨集合共通欄，如村莊/季節/賣價）或 `detailColumns`
（條目頁限定欄，如角色卡的「喜歡的服裝」「約會時段」），也不論值是單值或陣列，
一律 `dt` 標題一行、`dd` 值換行置於下方（左對齊），虛線格線不變。
（2026-07-19 曾有「單值左右並排／多值上下」兩變體並存的舊決策，已被本次裁決取代。）

**家庭關係（family）再變體**（C11，2026-07-19）：值本身是「稱謂＋可點角色連結」
（如「母親：傑西卡（ジェシカ）」），無法用 chips（chips 不支援前綴文字）也無法用
`formatColumnValue`（純字串，不含連結），改在資訊列內自訂渲染——
每行一個 `稱謂：<a>中文（日文）</a>`；查無站內角色時降級純文字（不猜）。

### 列表卡（EntryCard）
`bg-(--village)/10` 圓角卡（村色 10% 透明度淡底色鋪滿卡片，`data-village` 決定
`--village` CSS 變數；U36，2026-07-22，取代舊「村色 2px 邊框」版——邊框保留給
互動當下的強調，常態的村莊歸屬改靠底色深淺分層）。名稱文字維持 `text-(--village)`
深色字，辨識度不靠邊框、靠色階。無特定村莊的卡片落在 `--color-plum`（見上方色盤，
與篩選 chip 的 ink 底色刻意不同色相，避免撞色）；卡片高度以 `h-full` 撐滿 grid
列格，內容較少的卡片底色仍撐到跟同排最高卡片齊平，不會矮一截。
標準版（`config.columns` 2 欄以上）：名稱（純中文，不帶日文）置頂，
下方一份 `dl` 逐欄 label＋值左右並排、`divide-dashed` 分隔線。

**單欄版（U28，2026-07-20）**：`config.columns` 只有 1 欄時（如 recipes 的
「名稱＋5★賣價」）改「名稱左、值右一行」——名稱與值同一行 `flex justify-between`，
不再另起 `dl` 區塊。名稱**帶日文**（「中文（日文）」，2026-07-20 使用者裁決），
對齊全站慣例；代價是長名稱（18 字含日文）在手機寬度可能換行，接受此外觀代價，
不為了不換行而犧牲日文名一致性。此版型不是 recipes 專屬特例，任何 collection
縮到單欄時同套用。

### 條目頁三語言分工（2026-07-19 使用者裁決）
條目頁只允許三種內容語言，職責固定，不混用：
- **欄位值 → 資訊列**（格線）：單值右對齊、多值用上述變體；
- **可點物品集合 → chips**：邊框＝可以點，不可點的多值（約會地點）不做成 chip；
- **敘事／時間序內文 → prose 手帳行**：條目頁 prose 的 `ul` 去圓點、每列點狀虛線底線、
  字級 `text-sm`（14px，2026-07-19 使用者裁決，與資訊列同語言同字級）；
  guide 攻略長文維持一般列點與預設字級，不套用。

### Chips（禮物等清單）
`bg-cream`、ink 細邊、不對稱圓角（`4px 10px 5px 9px`，撕紙感）、細硬陰影。
「最愛」用 seal 紅邊紅字；「討厭」ink 55% 加刪除線；
「最討厭」（loathes）＝ seal 紅邊紅字＋刪除線——「最」級用 seal 紅、負面用刪除線，兩軸疊加。
**這裡的邊框是「可以點」的語意**（本節指 `ItemChips`，角色頁禮物清單 loves/likes/
hates/loathes；條目頁三語言分工「可點物品集合 → chips：邊框＝可以點」同一條規則）。

**篩選 chip（FilterBar/ChipGroup，U36，2026-07-22）與上述 ItemChips 是不同語意，
不可互套**：篩選 chip 表達的是「選中狀態」而非「可點連到別處」，改成無邊框實色——
未選 `bg-ink/8`、已選 `bg-ink text-parchment`，拿掉描邊；「篩選 N ▼」開關鈕同套。
篩選面板本身不再外包一層框（拿掉舊 `rounded-2xl border-2 border-dashed` 大盒子），
純間距排列讓 chip 群組直接呼吸，避免「框中框」。

### 搜尋框
無邊框、底部 `2px` 虛線（ink 45%）、左側放大鏡線條圖示；focus 時虛線轉實線加深。
**focus 回饋只靠底線變化，input 本身不出現 outline 外框**（2026-07-15 使用者裁決）——
是「互動元素要有 focus-visible 外框」的唯一例外。**CollectionPage 與 Home 共用同一套
底線規格**（U36，2026-07-22 修正走鐘——CollectionPage 一度誤用整圈圓角邊框，
與此處規格不一致），不得再出現整圈邊框變體。

### 緞帶列（ribbon）
全寬入口列（如行事曆）：`bg-soil/15`、虛線邊框、圓角、左圖示右箭頭。
**多條緞帶列直向堆疊**（U18，2026-07-19）：新入口沒有對應貼紙格時，加一條緞帶列而非
硬塞進貼紙網格——貼紙網格行動 3×3／md 5×2 都已剛好填滿，插入會破格數；緞帶列零版面
衝擊，各條在所有尺寸皆顯示（不像行事曆緞帶只在行動版顯示、桌機併入貼紙格——因為新入口
沒有貼紙格可併）。

### 攻略總覽頁（guides index，U18，2026-07-19）
`guides` collection（文章目錄形態）**不併入** `CollectionPage`/`collectionConfigs` 機制
（欄位卡/篩選機制在純目錄頁無用武之地）。版型：依 `system` 分組，每組一個印章圖示＋
中文標籤（見下方 `SYSTEM_LABELS`）＋篇數小字當組標題，組內條目用資訊列同款的點狀虛線
行（`border-b-[1.5px] border-dotted`）列出標題可點連結，不分二層子分類（life 22 篇即使
逾半是任務攻略，也維持平面列表，不另立「任務／事件」子分組——顆粒度用不到）。
`system → 中文標籤／印章圖示` 全站唯一對照表：

| system | 標籤 | 圖示 |
|---|---|---|
| basics | 基礎 | compass |
| farming | 農耕 | wheat |
| livestock | 畜牧 | sheep |
| cooking | 料理 | pot |
| fishing | 釣魚 | fish |
| bugs | 昆蟲 | bug |
| mining | 採礦 | gem |
| life | 生活 | house |
| romance | 戀愛 | heart |

`compass`／`house` 為本次新增圖示（`icons.jsx` sprite，24×24、stroke 1.8，與既有圖示
同規格）；其餘 7 個沿用 collection 貼紙已用的圖示（跨 collection/guide 共用同一視覺
語彙）。首頁緞帶列的「攻略總覽」入口另用新增的 `book` 圖示（代表「文章目錄」，與 9 個
system 圖示區隔）。`GuidePage` 頂部加「← 攻略總覽」回鏈（回 `/guides`，與全域「← 回上
一頁」語意不同——後者是瀏覽器歷史返回，前者是固定的目錄入口，深連結直開時仍可用）。

## 版面原則

- 手機優先：內容寬 `max-w-lg` 置中，`bg-parchment bg-dots` 滿版鋪底。
- 主內容容器：cream 面、ink 細邊、硬陰影（與貼紙同語言）。
- 密度低：卡片間距 `gap-3` 起跳，不塞滿。
- 長表格在自己的容器內 `overflow-x-auto`，頁面本體不得橫向捲動
  （C12：角色條目頁「戀愛事件」段落表格比照 guide 頁套用同一組 `[&_table]`/`[&_td]`/`[&_th]` 護欄）。

## 深度與高度

陰影一律**硬陰影**（offset、零 blur、ink 低透明度）——像紙疊在紙上；
不用柔和發散陰影（拍立得是唯一例外）。層級靠邊框與底色區分，不靠陰影深淺。

## Do's and Don'ts

- ✅ 圖示一律走 `icons.jsx` sprite＋印章框；新圖示先加進 sprite。
- ✅ 旋轉角度只用既定檔位（±0.5–1.4°；印章 ±4–5°），統一從 CSS 類取得。
- ✅ 互動元素要有 `focus-visible` 可見框（ink 2px outline）。
- ❌ emoji 當圖示或章節記號。
- ❌ 楷體排內文長段落。
- ❌ 純白底、冷灰、藍紫漸層。
- ❌ 柔陰影（拍立得除外）、發光效果。
- ❌ 在 seal 紅之外再引入新的強調色。

## 響應式行為

- 基準寬 375px；破版檢查跑 375 / 768 / 1280 三檔（同 T8.3）。
- **階梯式加寬**（2026-07-15 使用者修訂原「殼固定 480–560px」決策）：
  殼寬 `max-w-lg → md:max-w-3xl → xl:max-w-5xl`；**不做桌機多欄版型**，
  維持單欄手帳結構，靠「格狀內容增欄」吃掉寬度——
  首頁貼紙牆 `3 → md:5` 欄（md 起行事曆併入貼紙牆湊滿 5×2，緞帶列只在行動版）、
  characters `3 → md:4 → xl:6` 欄、其他 collection 列表 `1 → md:2 → xl:3` 欄。
  長表格（guide 內容）隨殼加寬自然獲得寬度，仍保 `overflow-x-auto` 護欄。
- 觸控目標 ≥ 44px（追蹤器按鈕尤其）。
- 微旋轉與紙膠帶為裝飾層，`prefers-reduced-motion` 不涉及（皆為靜態）；
  若日後加動畫（蓋章、翻頁），須尊重 `prefers-reduced-motion`。
